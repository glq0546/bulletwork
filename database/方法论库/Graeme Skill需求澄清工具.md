# Graeme Skill需求澄清工具

**日期**：2026-06-13

## 核心内容

Graeme Skill用于在需求不清、任务范围模糊、容易做偏的情况下，先把问题澄清再动手。它的价值不是给答案，而是把“我要做什么、为什么做、做到什么程度算完成”问清楚。

## 适用场景

- 用户只说“帮我优化”“帮我做一个页面”“帮我分析一下”，但没有明确目标。
- 项目存在多个可选方向，需要先收敛。
- 涉及产品定位、文案、功能范围、发布策略、商业模式。
- 一旦做错会浪费大量时间的任务。

## 标准澄清框架

1. **目标**：这件事最终要达成什么结果？
2. **用户**：服务谁？谁会使用或阅读？
3. **场景**：用户在什么情况下需要它？
4. **约束**：时间、成本、技术、合规、平台规则是什么？
5. **成功标准**：做到什么程度算完成？
6. **不做什么**：哪些范围明确排除？
7. **下一步**：最小可执行动作是什么？

## 使用方法

先让Graeme提出3–7个关键问题，不允许一次问十几个泛泛问题。每个问题必须能改变后续方案。如果答案不影响决策，就不问。

## 输出格式

```text
我理解的目标：
关键不确定点：
必须澄清的问题：
建议默认假设：
最小下一步：
```

## 项目经验

在一人公司模式下，需求澄清能避免AI直接开干导致方向错误。CEO负责拍板，AI合伙人负责提出关键问题和默认建议。Graeme Skill尤其适合在产品立项、功能边界、内容策略和推广动作前使用。
