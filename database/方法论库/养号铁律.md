# 养号铁律

**日期**：2026-06-13

## 核心内容

本文件记录Reddit、X/Twitter、Indie Hackers养号规则、Karma门槛和Chrome档案隔离策略。结论：**新号先做人，后推广；先互动，后发链接；一个身份一套环境。**

## 一、总原则

新号前5–7天不发推广链接；Karma低于20分前不做任何明显推广；一个设备/浏览器档案只登录一个项目身份；不要频繁切换新老号；不要复制粘贴同一套话术到多个平台；先贡献回答，再提产品；推广内容比例不超过10%。

## 二、Reddit铁律

Karma 0–5：只点赞、收藏、读帖，不发链接。Karma 5–20：评论帮助别人，不能提产品。Karma 20–50：可在高度相关场景软提，不放主帖链接。Karma 50+：可发求反馈帖，链接放评论区。

禁止动作：新号当天发产品链接；Karma不足时去r/SideProject硬推；同一IP/浏览器频繁切换多个Reddit号；使用AI味明显的评论；多账号互相点赞；被封后立刻用同环境注册新号。

推荐动作：每天5–10分钟真实互动；评论先回答问题，最后才轻提工具；链接优先放评论，不放主帖正文；多参与目标板块之外的普通讨论，让账号像真人。

## 三、X/Twitter养号规则

新号前3天不要频繁发外链；先完善头像、简介、置顶帖；每天发1条Build in Public内容；每天回复5条目标用户或独立开发者内容；私信只用于少量真实求反馈，不群发。

可发内容：我做了一个工具但没人用，正在找原因；今天修了支付/部署/SEO问题；从Reddit封号学到的推广教训；做ChargeDecode时发现的信用卡账单痛点。

## 四、Indie Hackers规则

IH适合故事，不适合硬广。标题要暴露真实困境，如「I built X and nobody is using it」。评论要共鸣和复盘，不要贴链接。发帖重点是过程、数据、失败、请求建议。产品链接只有在别人问时再给。

## 五、Chrome档案隔离策略

子弹-周报项目推广：只登录BulletWork相关X/IH/未来Reddit。解码-信用卡项目推广：只登录ChargeDecode相关Reddit/X/Gumroad。默认档案：日常浏览，不登录Reddit。不同项目账号不要混登；被封账号所在档案不要继续注册同平台新号；Cookie、密码、插件环境分开；项目邮箱、Chrome档案、社媒身份一一对应。
