# 三轮外部审计交叉比对项目体检方法

**日期**：2026-06-13

## 核心内容

三轮外部审计交叉比对法用于对已上线项目做系统体检。核心思想是：不要只相信一个AI或一个视角，而是用不同模型、不同角色、不同审查维度交叉验证，最后由CEO拍板。

## 一、三轮审计结构

### 第一轮：技术与安全审计

重点检查：HTTPS、CSP、安全响应头、API Key暴露、CORS、支付SDK、路由、sitemap、robots、页面404、移动端显示、性能阻塞。

适合角色：Troy Hunt视角、Addy Osmani视角、克劳扣得代码审查。

### 第二轮：内容与SEO/GEO审计

重点检查：Title、Meta Description、H1、FAQ Schema、Article Schema、GEO引用块、内链、CTA、AI套话、事实准确性、发布日期、合规声明。

适合角色：谷歌AI Studio、智谱GLM、SEO顾问视角。

### 第三轮：商业与转化审计

重点检查：用户是否一眼看懂价值；CTA是否清晰；免费内容和付费产品分工是否明确；定价是否能解释ROI；页面是否建立信任；是否有下一步动作。

适合角色：俞军、梁宁、Naval、Paul Graham、芒格视角。

## 二、交叉比对方法

每一轮都要求输出：问题、证据、严重度、建议修复、是否必须立刻修。然后把三份结果合并成一张表。

优先级规则：

1. 安全/支付/访问错误优先。
2. 影响Google抓取和页面收录的问题优先。
3. 影响用户理解和购买的问题优先。
4. 纯美化、低影响优化延后。

## 三、避免误判

如果一个模型指出问题，另一个模型未提到，不直接认定为真。必须回到源码、线上页面或GSC/GA4/Cloudflare数据验证。AI审计只能发现线索，不能替代最终检查。

## 五、项目经验

ChargeDecode全面体检时发现过GEO引用块缺失、Footer年份错误、AI套话残留、sitemap抓取优先级低、首页内链不足等问题。三轮审计能把“看起来已完成”的项目重新拉回可验证状态。
