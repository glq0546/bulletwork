# 核心商业逻辑

**日期**：2026-06-13

## 核心内容

本文件汇总项目过程中反复验证过的商业逻辑。核心方向不是做大而全的平台，而是用AI和内容资产，在低成本条件下快速验证「真实痛点 → 可交付资产 → 可变现路径」。

## 一、总逻辑：一人公司不是拼资源，而是拼验证速度

当前资源结构：CEO有时间、执行力、决策力；AI合伙人负责技术、策略、内容、风险预警；克劳扣得负责代码和部署；谷歌AI Studio负责英文内容和事实搜索；智谱GLM负责交叉检查；龙猫负责批量生成和提炼。所以商业模式必须不依赖大团队、不依赖复杂销售、不依赖高预算广告、能快速上线、能把一次劳动沉淀成长期资产。

## 二、内容站 + 数字产品是当前最优路径

ChargeDecode验证了这条路径：免费文章解决用户即时困惑「这笔扣款是谁」；文章通过SEO/GEO获取长期流量；CTA引导到Pro PDF；Pro PDF解决更深问题「怎么退款、怎么争议、怎么审计订阅」；数字产品边际成本接近零，适合一人公司。

## 三、免费内容和付费产品的分工

免费文章解决「这是什么」，获得流量和信任；免费工具解决「能不能帮我生成/判断」，获得试用和邮箱；付费PDF解决「我该怎么做」，交付模板、清单、指南；订阅工具解决「我想长期省时间」，形成SaaS收入。

## 四、价值翻译逻辑

用户不为功能付费，用户为结果付费。错误表达：100+ merchant code dictionary、AI weekly report generator、5 dispute letter templates。正确表达：帮你识别账单上看不懂的扣款，减少误判和漏退；把15分钟周报压缩到60秒；直接复制英文争议信，提高追回费用的概率。

## 五、从推式推广转向拉式推广

旧策略是私信陌生人、冷邮件、硬发链接，结果是Reddit封号、冷邮件退信、LinkedIn限制、用户防御心强。新策略是Build in Public + SEO + 内容站：分享过程不像广告，一篇内容可长期被搜索，失败记录也能成为信任资产，适合不擅长社交但执行力强的CEO。

## 六、项目选择逻辑

优先选择用户已经在搜索、有强情绪、能用PDF/模板/工具交付、能用SEO长期获客、风险可控的问题。暂不选择需要金融牌照的建议型产品、复杂B端销售、大量人工服务的Agency模式、Notion/Canva通用模板这类竞争重且差异弱的产品。

## 七、海外软件目录上架门槛认知

Product Hunt、G2、AlternativeTo、SaaSHub、Capterra、TrustRadius、Slant.co等目录不是同一种渠道。Product Hunt偏发布事件，G2偏B2B信任背书，AlternativeTo偏替代品搜索，SaaSHub偏工具目录，Capterra/TrustRadius更偏企业软件评价。

早期产品最大误区是把目录当成流量来源。实际情况是：很多目录需要审核、真实用户评价、公司信息、截图、分类匹配，甚至付费或达到用户数量门槛。没有真实用户时，上架只能增加可信度，不能指望立刻带来转化。

认知结论：海外软件目录适合做“信任资产”和SEO外链，不适合做早期唯一获客渠道。优先级应低于内容站、SEO、Build in Public和真实用户反馈。

## 八、GA4与Cloudflare Analytics分工认知

GA4适合看用户行为：来源、页面访问、事件、转化漏斗、CTA点击、留存趋势。Cloudflare Analytics适合看基础访问和技术层数据：请求量、地区、缓存、状态码、机器人访问、边缘网络情况。

两者不能互相替代。GA4回答“用户做了什么”，Cloudflare回答“网站被访问和服务得怎么样”。早期如果GA4没有数据，不一定代表没人访问，也可能是跟踪代码、Cookie、拦截器或事件配置问题；如果Cloudflare有请求但GA4没有用户，需要检查前端埋点。

认知结论：产品增长看GA4，站点健康看Cloudflare，两者结合才能判断是“没人来”“埋点坏了”还是“来了但没转化”。

## 九、避免凌晨集中登录多个网站

凌晨集中注册、登录、验证多个海外网站，会让谷歌账号、社媒账号、软件目录账号更容易触发异常风控。尤其是同一时间段内连续打开多个新平台、验证邮箱、切换IP或Chrome档案，会形成非正常用户行为特征。

处理原则：账号注册和目录上架分散到白天或多个时间段；一个账号一次只做少量关键动作；验证邮箱后不要立刻连续注册下一批；重要账号开启2FA和辅助邮箱；不同项目身份使用不同Chrome档案。

认知结论：账号安全也是商业基础设施。为了快一点注册而触发风控，损失远大于节省的时间。

## 十、核心结论

当前最适合的赚钱路径是：**痛点雷达发现问题 → 内容站承接搜索 → 免费文章建立信任 → PDF/模板变现 → 数据反馈决定是否做工具。**

## 我们的客户不是"随便看看的人"
**日期**：2026-06-12
**来源**：关于"赚谁的钱"的战略讨论
**核心认知**：赚想赚钱的人的钱，本质上是在卖"杠杆"——时间杠杆（SOP合集）、信息杠杆（数据简报）、效率杠杆（提示词库）、确定性杠杆（解码Pro版）。和赚普通消费者的钱有本质区别：决策心理是"投资"而非"消费"，付费意愿更强，用户粘性更高，客单价天花板更高（$19.99-99+ vs $5-20）。
**落地行动**：所有产品描述从功能型改为ROI型，统一加上"This helps you make/save money by..."

## 解码项目的核心用户画像：不是“所有人”，是“正在搜索不明扣款的人”
**日期**：2026-06-04
**类型**：判断
**核心内容**：讨论目标用户时，CEO反复追问“用户是谁”。最终锁定的精准画像是：刚发现一笔不明扣款、正在Google搜索“what is this charge”的人。不是“所有信用卡用户”，不是“经常出国的人”，不是“财务管理者”。这个精准锁定直接决定了获客路径——SEO截流长尾商户名搜索词。
**价值点**：用户画像的精准度决定获客效率。很多项目失败是因为“目标用户”太宽泛，无法有效触达。

## PDF定价逻辑演变：从$4.99冲动消费价到$19.99工具价值价
**日期**：2026-06-09
**类型**：创意
**核心内容**：早期讨论定价$4.99，认为是冲动消费价。但经过分析发现，用户搜到我们文章时已经知道“这笔钱不该付”，有明确的追回预期。这时候$19.99只要低于预期追回金额（通常$50-$100），转化逻辑就成立。从“冲动消费价”升级为“工具价值价”。CEO原话是“花钱少办大事”——定价需要让用户觉得少花钱能办大事。
**价值点**：数字产品的定价应该基于用户预期的ROI，而不是行业平均价格。帮用户找回$100的工具，定价$20比定价$5更有说服力，因为价格太低反而让用户怀疑效果。
