# 入库指南

本文件用于记录 database 各库的用途、识别标准、入库模板与安全规则。具体内容稍后填充。

## 安全规则
1. 绝对禁止将数据库内容上传到任何云端服务
2. 绝对禁止在公开对话中透露API Key和密码
3. 绝对禁止将账号清单发送给任何第三方
4. API Key和密码只从本地配置文件或桌面密码记事本中读取，不在数据库内存放

# 一人公司数据库 · 入库指南（最后更新：2026年6月13日）

## 数据库结构总览
本数据库包含20个子目录，覆盖一人公司全部资产类型。

## 各库用途与识别标准
- **方法论库**：SOP、工作流程、操作标准。识别：看到"第一步第二步第三步"或"标准流程"→放这里
- **数据资产库**：痛点数据、调研报告、竞品分析。识别：看到"JSON数据""表格""统计"→放这里
- **认知库**：商业逻辑、推广话术、变现策略。识别：看到"核心观点""启发""策略"→放这里
- **经验库**：踩坑记录、技术攻坚、教训。识别：看到"被封""报错""失败""解决"→放这里
- **产品库**：产品矩阵、定价策略、价值翻译模板。识别：看到"产品""定价""变现"→放这里
- **产品PDF库**：实际PDF产品文件。识别：看到".pdf"且是最终售卖品→放这里
- **账号资产库**：所有平台账号清单。识别：看到"用户名""邮箱""密码"→放这里（不记录密码原文）
- **素材库**：Logo、截图、推广图片。识别：看到".png""截图""封面"→放这里
- **文章库**：碎片、草稿、灵感。识别：未成型的文字片段→放这里
- **内容资产库**：已发布/待发布的完整文章。识别：已上线的文章成品→按项目分目录放
- **代码库**：常用指令、脚本。识别：看到"代码""命令""脚本"→放这里
- **调研库**：竞品分析、市场报告。识别：看到"竞品""市场""分析报告"→放这里
- **项目档案库**：每个阶段的发展历程。识别：看到"阶段总结""里程碑"→放这里
- **用户反馈库**：Gumroad评价、Reddit/X回复。识别：看到"用户说""评价""反馈"→放这里
- **合作伙伴库**：信用卡部、DeepSeek窗口分工。识别：看到"分工""协作""联系方式"→放这里
- **数据追踪库**：GA4/Search Console历史数据。识别：看到"流量""排名""点击"→放这里
- **失败记录库**：已验证后放弃的方向和渠道。识别：看到"放弃""不再使用""验证失败"→放这里
- **工具配置库**：所有AI工具和平台的配置信息。识别：看到"API Key""配置""接入方式"→放这里（不记录Key原文）
- **法律合规库**：审查结论、免责声明模板、合规标准。识别：看到"合规""免责声明""法律"→放这里
- **视频文案库**：视频提炼分析。识别：看到"视频链接""核心观点"→按领域放对应子目录
- **提示词库**：验证过的提示词（按L1-L3分级）。识别：看到"提示词""Prompt"→按用途放对应子目录

## 文件命名规范
统一用 `主题-日期-版本.md` 格式。示例：`三层搜索SOP-20260613-v1.md`

## 入库模板
### 方法论类
# [方法论名称]
- **制定日期**：2026-06-XX
- **最后更新**：2026-06-XX
- **核心流程**：（1-2-3步骤）
- **适用场景**：（什么时候用）
- **前置条件**：（需要什么工具/数据）

### 认知类
# [认知主题]
- **来源**：视频/对话/实战
- **日期**：2026-06-XX
- **核心观点**：（1-2句话）
- **和我们的关系**：（对项目的启发）
- **落地行动**：（接下来做什么）

### 经验类
# [踩坑记录]
- **日期**：2026-06-XX
- **问题描述**：（发生了什么）
- **根因**：（为什么会发生）
- **解决方案**：（怎么修好的）
- **教训**：（下次怎么避免）

### 产品类
# [产品名称]
- **定价**：$X.XX
- **状态**：已上架/待打包/规划中
- **目标用户**：（谁需要它）
- **价值翻译**：（用户为什么付费）
- **更新日志**：（每次修改记录）

## 安全规则（必须遵守）
1. 绝对禁止将数据库内容上传到任何云端服务
2. 绝对禁止在公开对话中透露API Key和密码
3. 绝对禁止将账号清单发送给任何第三方
4. API Key和密码只从本地配置文件或桌面密码记事本中读取，不在数据库内存放
5. L3等级的提示词绝对禁止打包售卖
6. 新AI打开数据库后，必须先读本文件，再读代号清单.md和项目当前状态.md
