# 三层搜索SOP提示词

## Prompt

你是一位独立开发者项目验证顾问。请用“三层递进搜索法”验证下面这个新项目方向是否值得投入。

**项目方向**：[填写项目方向]

**目标用户**：[填写目标用户]

**初步产品形态**：[网站 / PDF / 模板包 / SaaS / 浏览器插件 / 数据简报 / 其他]

**我的约束条件**：

- 单人开发。
- 低成本。
- 希望一周内做出MVP。
- 不做复杂B端销售。
- 尽量靠SEO、社区和数字产品变现。
- 需要规避法律、金融、隐私和平台风控风险。

## 第一层：公开搜索层

请生成并执行/模拟至少20组搜索关键词，用于确认这个问题是否真实存在。

关键词必须覆盖：

1. 问题型关键词。
2. 痛苦表达型关键词。
3. 结果导向型关键词。
4. 竞品型关键词。
5. 平台限定关键词。

示例格式：

- “why [problem]”
- “how to [solve problem]”
- “I hate [workflow]”
- “best [tool/template] for [problem]”
- site:reddit.com “[pain phrase]”
- site:github.com “[topic]” “issue”

输出本层结论：

- 搜索需求是否明确。
- 用户主要搜索什么。
- 是否有SEO机会。
- 是否已有竞品或付费产品。

## 第二层：社区痛点层

请从Reddit、X/Twitter、Indie Hackers、Hacker News、GitHub Issues/Discussions、Quora等公开讨论中寻找真实用户抱怨。

每条有效痛点必须记录：

- 用户原话或接近原话。
- 来源平台。
- 用户画像。
- 情绪强度：高/中/低。
- 具体损失：钱、时间、风险、焦虑、麻烦。
- 现有解决方案是否让用户不满意。
- 是否出现“我愿意付费”“有没有工具”“有没有模板”等信号。

过滤掉：

- 新闻转载。
- 品牌宣传。
- 纯教程文章。
- 没有具体用户经历的内容。

输出本层结论：

- 是否存在真实痛点。
- 痛点是否重复出现。
- 用户是否有强情绪。
- 最典型的3条用户原话是什么。

## 第三层：商业验证层

请验证这个方向能否变成钱。

检查：

1. Gumroad、Etsy、App Store、Chrome Web Store、Product Hunt、SaaSHub、G2上是否有类似付费产品。
2. 竞品价格区间。
3. 用户是否愿意为结果付费。
4. 是否能做成内容站、PDF、模板包、SaaS、插件或数据简报。
5. 是否能用SEO长期获客。
6. 是否有合规风险。
7. 一周内能否做出MVP。

输出本层结论：

- 最适合的产品形态。
- 建议定价。
- 最小MVP版本。
- 首批文章/页面/模板清单。
- 不建议做的部分。

## 六条标准评分

请按以下六条标准打分，每项1–5分，并给出判断依据：

1. 痛点强度。
2. 付费意愿。
3. 单人可执行。
4. 获客路径清晰。
5. 合规与风险可控。
6. 资产复用能力。

总分解释：

- 24–30：立即进入MVP或变现测试。
- 18–23：继续补数据，先做内容验证。
- 12–17：放入储备库。
- 12以下：放弃或归入失败记录库。

## 最终输出格式

请用中文输出：

1. 一句话结论：做 / 暂缓 / 放弃。
2. 三层搜索发现摘要。
3. 真实痛点证据表。
4. 竞品和变现证据表。
5. 六条标准评分表。
6. 建议MVP。
7. 建议SEO关键词。
8. 建议Gumroad/PDF/模板产品。
9. 最大风险。
10. 下一步72小时行动清单。

要求：直击要害，不要泛泛而谈。如果证据不足，明确说“证据不足”，不要为了显得完整而编造。
